Model Wizard Overview

Introduction

The Model Wizard will do two things. Watermark removal and custom model preparation.

Default workflow

Consider the following workflow:

Loading a model downloaded from the blace.ai hub into your application will result in watermarked outputs. E.g. images have a cross overlaid and text is slightly mangled:

Watermarked output from demo application

Model unlocking

This mode will remove the trial flag from the model and therefore your applications outputs are watermark-free:

It will be invoked like
.\model_wizard unlock depth_anything_v2_small_v10.h depth_anything_v2_small_v10_unlocked.h
Afterwards you can replace all
#include depth_anything_v2_small_v10.h
in your code with
#include depth_anything_v2_small_v10_unlocked.h
and the outputs will be watermark free.

Model from custom models (torchscript and onnx)

In order to provide your custom models for consumption by blace.ai you need to pass it through the model wizard. This will enrich the model with meta data and encrypt / obfuscate the raw model payload, so it is protected once deployed to third parties:

The model meta is a .json file describing some model characteristics like inputs and outputs. This will also write the obfuscated model payload to disk.

Afterwards you can include the model:
#include custom_model.h

See the Model Wizard Creation guide for a detailed description.
